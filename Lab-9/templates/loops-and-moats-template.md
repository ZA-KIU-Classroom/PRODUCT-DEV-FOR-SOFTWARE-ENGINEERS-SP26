### K-Factor Estimate

```
K = invitations sent per user per month x conversion rate of invitations
```

| Input | Value | Source |
|-------|-------|--------|
| Average invitations sent per user per month | [Number] | [Our data / benchmark / assumption] |
| Conversion rate of those invitations | [X%] | [Our data / benchmark / assumption] |
| **K-factor** | **[invitations x conversion rate]** | |

**Arithmetic shown:**
K = [invitations per user] x [conversion rate] = [result]

**What our K-factor means:**

If K is less than 1: Each cohort of users brings in fewer new users than itself. The loop slows decay but does not compound. Our loop still matters because it reduces our blended CAC by approximately [1 / (1 - K) x 100]%. At K = [our value], our effective CAC is reduced by approximately [calculated reduction]%.

If K equals 1: Each cohort exactly replaces itself. Stable but not compounding.

If K is greater than 1: Each cohort is larger than the previous. Compounding growth.
