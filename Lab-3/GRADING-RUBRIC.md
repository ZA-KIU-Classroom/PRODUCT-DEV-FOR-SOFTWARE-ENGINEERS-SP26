## Team Deliverable Quality (informs Checkpoint 1 assessment)

### Interview Logs

| Level | Descriptor |
|-------|-----------|
| Strong | Each log contains: participant profile with ICP fit assessment, 3+ verbatim quotes in quotation marks, key observations including non-verbal or emotional cues, preliminary insights phrased as hypotheses, open questions. Log clearly distinguishes between what was said and what the team inferred. |
| Adequate | Logs present and committed. Quotes present but some paraphrased rather than verbatim. Observations thin. Insights exist but stated as facts rather than hypotheses. |
| Weak | Bullet-point summaries without verbatim quotes, or no distinction between participant words and interviewer interpretation. No observations section. |
| Not present | No logs committed by the deadline. |

### Affinity Map

| Level | Descriptor |
|-------|-----------|
| Strong | Screenshot committed. Cluster names are problem statements, not category labels. Notes file identifies the unexpected cluster and directly addresses whether the map confirmed or contradicted the original hypothesis. Preliminary problem statement is evidence-grounded. |
| Adequate | Screenshot committed. Clusters named but names are category labels. Notes file present but pattern analysis thin. |
| Weak | Affinity map attempted but not committed, screenshot unreadable, or clusters vague and unlabeled. Notes file missing. |
| Not present | No affinity mapping evidence committed. |

### Interview Script v2

| Level | Descriptor |
|-------|-----------|
| Strong | Changelog documents which questions were dropped, added, or rephrased with specific evidence from interviews. New questions are open-ended, past-tense, and do not mention the solution. Script is tighter than v1. |
| Adequate | v2 exists and committed. Some changes made but changelog absent or vague. Some questions still hypothetical or slightly leading. |
| Weak | v2 nearly identical to v1. No visible reflection on what interview data revealed about the script. |
| Not present | No v2 committed. |

| Level | Descriptor |

## Common Mistakes to Catch in Real Time

1. **Summarizing instead of quoting.** "The participant said it was frustrating" is not a quote. Send them back to find the exact words.

2. **Affinity cluster names are category labels.** "Communication" or "Time management" are not useful. Push toward problem statements: "Students cannot get a clear answer without waiting 48 hours."

3. **Script v2 introduces solution questions.** "Would you use a feature that does X?" is a Mom Test violation. Catch this during the script workshop.

4. **ICP drift.** A recruited participant does not match the ICP and was logged without noting the mismatch. The ICP fit assessment field in the log template is there to catch this.

5. **One person driving all affinity mapping.** Assign explicit roles: one person per cluster to own naming, all members extracting stickies independently in Step 1.
