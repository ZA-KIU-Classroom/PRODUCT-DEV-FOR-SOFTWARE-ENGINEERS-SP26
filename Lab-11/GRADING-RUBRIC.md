## Activity 1: Competitive Matrix (`06-strategy/competitive-analysis.md`)

### Scope and coverage

| Level | Description |
|-------|-------------|
| Strong | 5+ competitors named specifically, including at least one indirect competitor and one substitute. 7+ dimensions evaluated. Competitors include well-known alternatives the user's target segment actually uses. |
| Adequate | 5 competitors present but all are direct competitors with no substitutes. 7 dimensions present but some are redundant or trivially similar. |
| Weak | Fewer than 5 competitors, or competitors are described generically without names. Fewer than 7 dimensions. |
| Missing | File absent, or fewer than 3 named competitors, or fewer than 5 dimensions. |

### Scoring honesty

| Level | Description |
|-------|-------------|
| Strong | Competitor scores reflect genuine assessment. Strong competitors score 4 or 5 on dimensions where they are genuinely strong. Own product does not score 5 on every dimension. |
| Adequate | Minor self-flattery present but core competitive dynamics are visible. |
| Weak | Own product scores 5/5 across all dimensions. All competitors score 0 or 1 on most dimensions. Matrix is not analytically useful. |
| Missing | No numeric scores present, or all entries are identical. |

### Synthesis paragraph

| Level | Description |
|-------|-------------|
| Strong | 200+ words. Names the specific Porter force representing the greatest threat with specific reasoning. Identifies two specific dimensions that create the most defensible gap, with explanation of why these dimensions matter to the target segment. Addresses what would have to change for the gap to close. |
| Adequate | Synthesis present but vague. Does not name a specific Porter force. Does not explain why the gap dimensions matter. |
| Weak | Fewer than 100 words. Generic statements without analysis. |
| Missing | Synthesis paragraph absent. |

## Activity 2: Strategy Canvas (`06-strategy/strategy-canvas.md`)

### Factor list and industry average

| Level | Description |
|-------|-------------|
| Strong | 8-10 factors listed. Industry average calculated from matrix scores. Own product scores included. Factors are specific to the market, not generic. |
| Adequate | 6-7 factors. Industry average estimated but not derived from matrix. Factors are reasonable but not market-specific. |
| Weak | Fewer than 6 factors. No industry average. Factors are generic. |
| Missing | File absent or fewer than 4 factors. |

### ERRC table

| Level | Description |
|-------|-------------|
| Strong | Every factor assigned to exactly one category. At least 2 factors in Eliminate or Reduce. At least 1 factor in Create that genuinely does not exist in the competitive landscape. Rationale column contains specific reasoning, not generic phrases. |
| Adequate | All factors assigned but skewed toward Raise. Only 1 factor in Eliminate/Reduce. Create factors describe improvements rather than genuinely new dimensions. |
| Weak | ERRC table present but all or most factors assigned to Raise. No genuine Eliminate or Create entries. |
| Missing | ERRC table absent. |

### Narrative

| Level | Description |
|-------|-------------|
| Strong | 150+ words. Explains the Blue Ocean move in plain language tied to the specific target segment. Names what was stopped and why. Names what was created and why the target segment values it. |
| Adequate | Narrative present but generic. Does not connect choices to the specific target segment. |
| Weak | Fewer than 75 words. |
| Missing | Narrative absent. |

## Activity 3: Ecosystem Map (`06-strategy/ecosystem-map.md`)

### Four categories populated

| Level | Description |
|-------|-------------|
| Strong | All four categories present (complements, partners, threats, complementors). Each category has at least 3 named entries. All entries are specific organisations, products, or named parties, not generic types. |
| Adequate | All four categories present. Some entries are generic. Complementors category is thin or misunderstood. |
| Weak | Fewer than 4 categories, or categories present but entries are entirely generic. |
| Missing | File absent or fewer than 2 categories. |

### Threats section quality

| Level | Description |
|-------|-------------|
| Strong | Each threat is named specifically. Likelihood rated for each (low, medium, high). Counter-strategy stated for each threat. Includes at least one non-obvious threat (platform, institutional, technology shift). |
| Adequate | Threats named but likelihood and counter-strategies missing. Only obvious direct competitors listed as threats. |
| Weak | Threats section present but entries are generic ("large tech companies", "future competitors"). |
| Missing | Threats section absent. |

### Strategic Priorities section

| Level | Description |
|-------|-------------|
| Strong | 150+ words. Names a specific organisation for the highest-priority partner relationship. Names a specific timeline. Names the specific highest-likelihood threat and a specific counter-strategy. Names a specific complementor and a plausible lightweight engagement approach. |
| Adequate | Strategic Priorities present but responses are generic. No specific organisations, timelines, or counter-strategies named. |
| Weak | Fewer than 75 words. Restates the categories without adding strategic reasoning. |
| Missing | Strategic Priorities section absent. |

## Activity 4: Moat Statement (`06-strategy/moat-statement.md`)

### Power named and mapped

| Level | Description |
|-------|-------------|
| Strong | One specific Helmer power named. The power selected is plausible given the product stage. The power is not branding, scale economies, or process power unless exceptionally well evidenced. |
| Adequate | Power named but the selection is not fully justified by the evidence. |
| Weak | Power named but the selection is implausible (branding claimed for a 13-week-old product with no supporting data). |
| Missing | No Helmer power named. |

### Mechanism explained

| Level | Description |
|-------|-------------|
| Strong | 2-3 sentences. Explains specifically how this power operates for this product, this user segment, and this market. Not a generic definition of the power. |
| Adequate | Mechanism explains the power generally but does not tie it to the specific product. |
| Weak | Mechanism restates the Helmer definition without applying it. |
| Missing | Mechanism absent. |

### Evidence quality

| Level | Description |
|-------|-------------|
| Strong | 3+ pieces of evidence, each linked to or cited from the repository. Evidence demonstrates the power is real today: analytics data, MOU documents, retention curves, engagement metrics, data source descriptions. |
| Adequate | 2 pieces of evidence, or 3 pieces that are not well tied to the claimed power. |
| Weak | 1 piece of evidence, or evidence consists of aspirational claims rather than current data. |
| Hypothesis (acceptable) | Document labelled "Moat Hypothesis." Evidence gap named clearly. Specific plan stated for collecting evidence before Demo Day. Earns partial marks. |
| Missing | No evidence and not labelled as hypothesis. |

### Path forward

| Level | Description |
|-------|-------------|
| Strong | 2-3 sentences with a specific, actionable plan for the next 60 days. Names specific actions (approaching named universities, building a specific feature, reaching a specific user milestone). |
| Adequate | Path forward present but vague. No specific actions or organisations named. |
| Weak | Fewer than one sentence. Generic aspiration without plan. |
| Missing | Path forward absent. |
